# CC-Remote v4 セットアップを実行してください（Claude Code 向け指示書）

あなたはこの PC に CC-Remote v4 をセットアップします。以下を上から順番に実行してください。

## 前提

- このフォルダに cc-remote-v4 プロジェクトがあること
- Node.js 18 以上がインストール済みであること
- ユーザーが信頼できる経路（GitHub Releases 推奨）から ZIP を DL 済み

## このツールの設計（受信者に伝える）

> CC-Remote v4 は完全 P2P で、中央サーバーがありません。配布者があなたのコマンドログを閲覧することは技術的に不可能です。
> 信頼点は 6 件のみ（README.md 参照）。配布者を信頼する必要はありません。

ユーザーがこの設計を理解した上で続けることを確認してください。

---

## 手順

### 1. 環境確認

```bash
node --version        # 18 以上
cloudflared --version # なければ winget install Cloudflare.cloudflared
```

### 2. 配布物検証（5 項目チェック）

```powershell
pwsh scripts/verify-package.ps1
```

5 項目すべて PASS が出ない場合、ユーザーに状況を伝えて停止してください。

### 3. setup.js を実行

```bash
cd <cc-remote-v4 のフォルダ>
node scripts/setup.js
```

setup.js が対話的に Step 0〜10 を実行します。各ステップでユーザーに確認しながら進めてください。

### 4. Step 0 で OneDrive パス警告が出た場合

- 警告内容をユーザーに見せる
- ユーザーに「展開先を OneDrive 外（C:\work\ 等）に移動して再度実行する」よう案内
- ユーザーが「このまま続ける」と判断した場合のみ y を入力

### 5. Step 9 はスキップ

v4 ではペアリング不要です。setup.js は自動的に Step 9 をスキップします。

### 6. Step 10 で PIN 設定が必要

PIN 仕様: **8 桁以上の数字**（例: 12345678）

**🚨 重要: Claude が自動で 00000000 等の値を入力しないこと。必ずユーザーが PIN を決めて入力してください。**

手順:
1. ユーザーに「ログイン用 PIN を 8 桁以上の数字で決めてください」と伝える
2. ユーザーが入力するのを待つ
3. Claude は input ストリームに自動入力しない

### 7. 起動後の QR コード

サーバー起動後、ターミナルにトンネル URL と QR コードが表示されます。
ユーザーに「スマホで QR を読み取って初回接続してください」と伝えてください。

### 8. 完了報告

ユーザーに「セットアップ完了。スマホからアクセスして PIN ログインしてください」と伝える。

## エラー対処

- `cloudflared not found` → `winget install Cloudflare.cloudflared`
- `Tunnel URL 取得タイムアウト` → ネットワーク確認、cloudflared 再インストール
- ポート 3737 使用中 → `netstat -ano | findstr :3737` で確認、該当プロセスを停止
- サーバー 10 秒以内に起動しない → ポート競合 or Node.js バージョン確認

詳細: [docs/TROUBLESHOOTING.md](TROUBLESHOOTING.md) 参照

---

## 注意事項

- 各ステップの実行結果をユーザーに見せながら進めてください
- エラーが出たら止まって原因を特定してから次に進んでください
- `pc.env` は秘密情報です。ログや出力に含めないでください
- **Step 10 の PIN 入力は絶対に Claude が自動入力しないこと**
- **不審に感じる動作があれば実行を止めて、ユーザーに確認すること**
